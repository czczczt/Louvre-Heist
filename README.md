## 1. Концепция

- **Название:** Louvre Heist
- **Жанр:** стелс-экшн, иммерсивный симулятор
- **Идея:** вор проникает в галерею чтобы похитить картину - нужно действовать скрытно, а эскалация насилия делает задачу всё сложнее
- **Фишка:** система классов влияет на доступные механики

---

## 2. Геймплей

- **Действия игрока:** движение, взаимодействие с объектами, диалоги, смена одежды, перенос тел, стрельба с глушителем / без, взлом
- **Цель:** украсть картину и добраться до одной из точек побега
- **Провал:** игрока поймали или убили
- **Смерть:** загрузить сохранение или начать заново

---

## 3. Классы

| Класс       | Ключевая механика                                                                                  |
| ----------- | -------------------------------------------------------------------------------------------------- |
| Красноречие | дополнительные варианты в диалогах, сможет убедить перенести картину в место, где ее легко украсть |
| Вор         | взлом замков, кража пропуска                                                                       |
| Насилие     | +количество пейджеров, легче убивает охранников                                                    |

---

## 4. Сущности

**Игрок:**

- здоровье, текущая одежда/роль, класс, инвентарь (пистолет, патроны, отмычки, пропуска)
- состояния: обычный, в маскировке, в розыске, необратимый лауд

**Охранники:**

- патрулируют маршруты
- реагируют на трупы, выстрелы, подозрительное поведение
- имеют пейджер - если не ответить в течение таймера, едет полиция
- в маскировке игрок свой, пока не делает подозрительных действий (достает оружие/открывает ящики столов или шкафы)

**Полиция:**

- приезжает после N тихих убийств или если на пейджер не ответили
- активно ищет игрока и не уезжает, пока не найдет

**Спецназ:**

- приезжает после убийства полицейских
- очень высокое здоровье, практически финальный сдерживающий фактор

**NPC (посетители галереи, персонал, владелец галлереи):**

- с некоторыми можно говорить
- могут дать подсказки, пропуск, изменить положение картины

---

## 5. Локация

- одна локация - галерея на основе реальной карты здания (сделать два этажа и подвал)
- несколько входов с разными требованиями (пропуск, взлом, диалог, вентиляция)
- закрытые зоны - нельзя заходить в чужой одежде без пропуска
- несколько точек побега (машина, другие выходы)

---

## 6. Системы

| Система             | Описание                                     |
| ------------------- | -------------------------------------------- |
| Движение + коллизии | базовое перемещение, стены, зоны             |
| ИИ охранников       | патруль, обнаружение, тревога, поиск         |
| Маскировка          | смена одежды, проверка поведения             |
| Диалоги             | дерево выборов, влияние класса на варианты   |
| Пейджер             | таймер после убийства, эскалация при провале |
| Эскалация           | охрана → полиция → спецназ                   |
| Инвентарь           | одежда, оружие, пропуска, инструменты        |
| Сохранения          | несколько слотов, JSON                       |

---

## 7. Интерфейс

- **Меню:** новая игра, загрузить, настройки (звук, управление)
- **Выбор класса** перед началом
- **HUD:** здоровье, текущая роль/одежда, уровень тревоги, инвентарь
- **Диалоговое окно:** портрет NPC, текст, варианты ответов
- **Меню паузы:** продолжить, сохранить, выйти

## Ядро (делать первым)

1. **Game loop + deltaTime** - основа всего
2. **AABB collision** - стены, зоны, объекты
3. **Camera offset + clamp** - камера следит за игроком на карте
4. **FSM (enum + switch)** - состояния игрока (обычный, маскировка, розыск, скрывается)

---

## ИИ охранников

5. **Patrol + LOS detection + hysteresis** - патруль, обнаружение игрока, задержка реакции
6. **A*** - охранники и полиция находят путь к игроку при тревоге
7. **Behavior tree** - сложное поведение охранника (патруль → подозрение → тревога → поиск)

---

## Игровые системы

8. **Observer / Event pattern** - пейджер, эскалация, триггеры диалогов
9. **Command pattern** - действия игрока (взлом, диалог, убийство) - пригодится для чистой архитектуры
10. **Weighted random** - вариативность патрулей, реплики NPC
